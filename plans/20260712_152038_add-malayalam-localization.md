# Add Malayalam localization + in-app language picker

**Status:** completed

## What the issue is

The app is English-only. `lib/l10n/` has just `app_en.arb` (about 570 message
keys). There is no Malayalam file and no way for a user to choose a language.
The app also does not follow the device language into Malayalam because no `ml`
locale exists.

The user wants:
1. A full Malayalam translation of every string.
2. An in-app language picker (System / English / Malayalam) in
   Settings › Appearance, where **System** follows the device language.

## Goal

- Malayalam works when the device is set to Malayalam (device fallback).
- The user can also force English or Malayalam from inside the app, ignoring the
  device language.
- The chosen language is remembered across app restarts.
- Nothing crashes on a missing translation (Flutter falls back to English per
  key automatically).

## Files to change

### New files
1. **`lib/l10n/app_ml.arb`** — full Malayalam translation. Same keys as
   `app_en.arb`, `"@@locale": "ml"`. Only the message values are translated; the
   `@key` metadata blocks are not needed in a translation file and will be
   omitted (the template file keeps them). Placeholders (`{name}`, `{count}`,
   plurals) are preserved exactly so ICU formatting still works.
2. **`lib/core/locale/app_locale.dart`** — small enum `AppLocale { system,
   english, malayalam }` with `label` (English fallback label), `prefValue`
   (stable string for storage), `fromPrefValue` (safe parse, defaults to
   `system`), and `toLocale()` returning `Locale?` (`null` for system).
3. **`lib/core/locale/locale_controller.dart`** — a Riverpod `Notifier<AppLocale>`
   mirroring `ThemeController`: synchronous hydrate from
   `keyValueStoreSyncProvider`, `setLocale()` writes through. Key
   `appearance.language`.

### Changed files
4. **`lib/l10n/app_en.arb`** — add 4 new keys for the picker:
   `appearLanguage`, `languageSystem`, `languageEnglish`, `languageMalayalam`
   (with `@` metadata). These also get translated in `app_ml.arb`.
5. **`lib/app.dart`** — watch `localeControllerProvider`; pass
   `locale: appLocale.toLocale()` to `MaterialApp` (null = follow device). The
   generated `supportedLocales` will already include `ml` once the ARB exists.
6. **`lib/shell/settings/sections/appearance_section.dart`** — add a language
   row (a `SegmentedButton<AppLocale>` or dropdown, matching the existing theme
   control) reading/writing `localeControllerProvider`. Labels come from the new
   l10n keys.
7. **`lib/l10n/app_localizations.dart`**, **`lib/l10n/app_localizations_en.dart`**,
   **`lib/l10n/app_localizations_ml.dart`** — regenerated by `flutter gen-l10n`.
   (`_ml` is new; the other two are regenerated with the 4 new keys.) These are
   generated, not hand-edited.

### Tests
8. **`test/l10n/localization_test.dart`** — add cases: `ml` is a supported
   locale; a few Malayalam strings resolve under `Locale('ml')`; a placeholder
   and a plural still format under `ml`.
9. **`test/core/locale/locale_controller_test.dart`** (new) — hydrate default =
   `system`; set/persist round-trip; unknown stored value falls back to `system`.

## Plan for the fix

1. Add the 4 picker keys to `app_en.arb`.
2. Create `app_ml.arb` by translating every value from `app_en.arb` into
   Malayalam, keeping all placeholders and plural syntax identical. This is a
   machine-assisted translation and will need a native-speaker proofread before
   release (I will note this in the change log).
3. Add `AppLocale` enum + `LocaleController` (copy the `ThemeController`
   pattern exactly for storage and hydration).
4. Wire `locale:` into `MaterialApp` in `app.dart`.
5. Add the language control to the Appearance settings section.
6. Run `flutter gen-l10n` to regenerate the localization Dart, then
   `flutter analyze` and `flutter test`.

## Notes / decisions

- **Sync:** the existing "Display settings" sync category is out of scope here;
  syncing the language choice can be a later change. I will not touch sync.
- **Theme/enum labels:** `AppThemeMode.label` is hardcoded English today; I will
  not change that in this plan (keeps the change focused on language). The
  language picker itself uses localized strings.
- **RTL:** Malayalam is left-to-right, so no directionality changes are needed.
- **Fallback safety:** if any `app_ml.arb` key is missing, Flutter uses the
  English value for that key, so the app never shows a blank or crashes
  (CLAUDE.md §3.4).
