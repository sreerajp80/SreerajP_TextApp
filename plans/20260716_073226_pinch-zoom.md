# Add Pinch-to-Zoom in Editor and Viewers

**Status:** completed

## The issue

Users currently cannot zoom in or out of the file text (editor or viewer surfaces) using touch gestures (pinch-in/out). 
Although there is an appearance font size setting slider in the settings screen, there is no gesture support on the main editor/viewer pages.

## Files to change

- `lib/core/theme/theme_settings.dart`
- `lib/shell/settings/sections/appearance_section.dart`
- `lib/core/editor/pinch_to_zoom_area.dart` [NEW]
- `lib/shell/tabs/tabs_workspace.dart`
- `lib/shell/tabs/degraded_document_view.dart`
- `lib/formats/csv/csv_grid.dart`
- `test/shell/tabs/pinch_to_zoom_test.dart` [NEW]

## The fix

1. **Theme Limits**: Update `ThemeSettings.minFontScale` to `0.5` and `ThemeSettings.maxFontScale` to `3.0` to support a wide range of zooming.
2. **Appearance Section**: Update settings page slider divisions to `25` so users can select sizes from 50% to 300% in 10% steps.
3. **PinchToZoomArea Widget**: Implement a wrapper widget that detects pinch scale gestures using a `GestureDetector`. When 2 or more fingers are active on the screen, it adjusts the font scale preference dynamically via the `themeControllerProvider`.
4. **TabsWorkspace**: Wrap `_DocumentBody` in `PinchToZoomArea` to enable gestures across all document formats.
5. **Degraded View**: Update `DegradedDocumentView` to fetch and use the provider's `fontScale` and `lineSpacing` when rendering `SelectableText`.
6. **CSV Grid Scaling**: Convert `CsvGrid` to a `ConsumerStatefulWidget`. Scale the row height, header height, row header width, and auto-fit column width computations based on the active `fontScale` to ensure cells and headers scale gracefully and never clip when zoomed in.
