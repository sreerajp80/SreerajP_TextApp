# Plan — Phase 7: CSV

**Status:** completed

Plan for building Phase 7 from [implementation-plan.md](../docs/implementation-plan.md)
(lines 462–511). This phase adds the **CSV** format module: a real table grid with
parsing and dialect detection, table navigation (freeze / sort / filter / jump), a raw
view toggle, read-only data insights (with a simple chart), full in-grid editing, and the
shared output / file-level operations (export, print, copy, metadata, split / merge).

It follows the same shape as the TXT (Phase 4) and Markdown (Phase 6) modules: **pure Dart
logic classes** that are unit-tested with no device, wrapped by **thin Riverpod / widget
UI** that only wires providers and gathers input. It reuses every Phase 3 (editor core:
encoding, atomic save, drafts, metadata) and Phase 5 (share / zip / print / export)
service unchanged, adding only a new `CsvExporter` to the existing export registry.

---

## 1. What this phase is

Phase 7 has six tasks:

- **7.1 Parsing & format handling** — detect the delimiter (comma / semicolon / tab /
  pipe); handle quoted fields containing commas and line breaks; detect encoding and line
  endings (reuse Phase 3 `TextCodecService`); first-row-as-header toggle; infer a column
  type (number / date / text / boolean / currency). Malformed rows never crash.
- **7.2 Table view & navigation** — a real grid with headers; freeze the header row and
  first column; hide / show columns; horizontal + vertical scroll; column resize / auto-fit;
  sort by header; filter / search rows; jump to a row; row / column counts; alternating row
  colors and grid lines.
- **7.3 Raw view toggle** — switch between the table and the raw delimited text.
- **7.4 Light data insights (read-only)** — per-column count / min / max / sum / average
  for numeric columns; unique / empty counts; a simple bar chart from a chosen column.
- **7.5 In-grid editing** — edit cells; add / delete / reorder rows and columns; edit the
  header; edit raw text in raw mode; duplicate-row detection / removal (whole row or a key
  column); undo / redo; find & replace; save preserves the detected
  delimiter / quoting / encoding / line endings (or a chosen set); Save / Save as a copy.
- **7.6 Output & file-level ops** — export CSV → PDF / JSON / HTML / XLSX; export
  selected / filtered rows only; print the table; copy cell / row / table; metadata (size,
  modified, row / col count, delimiter, encoding); split by row count / size (header
  repeated) / merge (same columns).

**User decisions already taken for this plan:**
- **XLS (legacy binary) is out of scope.** The plan text says "XLSX(+XLS)"; we ship
  **CSV → XLSX only** (hand-built OOXML on `archive`, same technique as the existing DOCX
  writer, fully open source per CLAUDE.md §3.1). The legacy `.xls` BIFF binary has no
  maintained open-source Dart writer and XLSX supersedes it everywhere.
- **The insights chart uses `fl_chart`** (MIT, open source).

## 2. What the issue is

The critical path is `… → 5 → (6|7|8|9) → 10 …`. Phases 5 and 6 are done. CSV files
already resolve to `DocumentFormat.csv` in `format_dispatch.dart`, but
`tabs_workspace.dart` only special-cases TXT and Markdown, so a `.csv` file opens as an
inert placeholder. `lib/formats/csv/` is an empty placeholder folder. Phase 7 fills it and
wires it into the shell.

CSV is a bigger module than TXT / MD in one way: its primary view is a **2-D data grid**,
not a text flow, and it is editable on both the grid side and the raw-text side. So the
session keeps a **table model as the source of truth** (not a single text string) and
serializes it to CSV text for the raw view, for save, and for the output services.

## 3. Package decisions (open source only — CLAUDE.md §3.1)

Three new packages, all open source; licenses confirmed before adding:

- **`csv`** (BSD-3-Clause) — RFC-4180 CSV parse and serialize (`CsvToListConverter`,
  `ListToCsvConverter`) with configurable delimiter, quote, and end-of-line. We do our own
  **delimiter detection** on top and pass the chosen dialect in.
- **`two_dimensional_scrollables`** (BSD-3-Clause, published by flutter.dev) — the official
  `TableView.builder` with **lazy row / column building and pinned (frozen) rows and
  columns**. This is what makes freeze-header / freeze-first-column and large-grid scroll
  work without Syncfusion, and it sets up Phase 10 virtualization.
- **`fl_chart`** (MIT) — the simple bar chart for task 7.4.

No banned or source-available SDKs. Syncfusion is not used anywhere.

## 4. Architecture — how it fits the existing code

Mirrors `MdDocumentSession` / `MdSessionManager` / `MdDocumentView` exactly:

- **`CsvDocumentSession`** (`ChangeNotifier`, no Riverpod) is the bridge between a shell tab
  and the views. It loads bytes via `SafService`, decodes via `TextCodecService` (never
  throws — CLAUDE.md §3.4), detects the dialect, parses into a `CsvTable`, tracks unsaved
  edits, drives crash-recovery drafts (`DraftStore` + `AutoSaver`), and saves through the
  `AtomicSaver` + `SafSaveTarget` preserving encoding + line endings + dialect
  (CLAUDE.md §3.5). It exposes the current content as `TextContent` for the shared output
  services.
- **`CsvSessionManager`** is the Riverpod provider owning one session per open CSV tab,
  keyed by tab id — a copy of `MdSessionManager`.
- The grid, raw view, toolbar, and sheets are **thin widgets** over the session and the
  pure logic classes.
- Save well-formedness: a CSV is always serializable, so the pre-save gate is trivially
  satisfied (rectangular table → valid CSV); no structured-gate is registered.

**Dual-edit model.** The `CsvTable` is canonical. The grid mutates the table directly. The
raw view is a `re_editor` surface bound to the serialized text; on leaving raw mode (or on
save) the raw text is re-parsed back into the table. Grid undo / redo uses a bounded
**snapshot** stack of the table (`CsvTableUndo`); raw-mode text editing uses `re_editor`'s
own undo / redo. `isDirty` = current serialized text ≠ saved text.

## 5. Files to change / create

### New — pure Dart logic (host-tested, no device)
- `lib/formats/csv/csv_dialect.dart` — `CsvDialect` value type (delimiter, quote char, line
  ending, `hasHeader`) + `detectDialect(sample)` (comma / semicolon / tab / pipe by scoring
  a sample outside quotes).
- `lib/formats/csv/csv_table.dart` — `CsvTable` model (header row + rows of `String` cells)
  with: serialize to CSV text (via `csv` `ListToCsvConverter`), `setCell`,
  `insert/delete/moveRow`, `insert/delete/moveColumn`, `renameHeader`, and
  `findDuplicateRows` / `removeDuplicateRows` (whole row or a key column).
- `lib/formats/csv/csv_parse.dart` — `CsvParse.parse(text, dialect)` → `CsvTable`; tolerant
  of ragged rows (pads / trims to the widest row); never throws (failure path returns a
  best-effort table or a friendly error).
- `lib/formats/csv/csv_types.dart` — `ColumnType` enum + `inferColumnType(values)`
  (number / date / boolean / currency / text).
- `lib/formats/csv/csv_insights.dart` — `CsvInsights.forColumn(...)`: count, min, max, sum,
  average (numeric), unique count, empty count.
- `lib/formats/csv/csv_split_merge.dart` — split by row count / size (header repeated on
  each part), merge files with the same columns; `merge(split(t)) == t`.
- `lib/formats/csv/csv_table_undo.dart` — bounded snapshot undo / redo of `CsvTable`.

### New — export (core/export)
- `lib/core/export/export_target.dart` — **extend** `ExportTarget` with `csv`, `json`,
  `xlsx`; add their `extension` / `mimeType` / `label` cases. (Additive; existing targets
  unchanged.)
- `lib/core/export/xlsx_writer.dart` — hand-built OOXML spreadsheet (zip of
  `[Content_Types].xml`, `workbook.xml`, `sheet1.xml`, rels) on `archive`, escaping cell
  text — same approach as `docx_writer.dart`.
- `lib/core/export/csv_exporter.dart` — `CsvExporter implements FormatExporter`
  (`formatId = 'csv'`); re-parses the CSV text it is given and builds PDF (table via `pdf`),
  JSON (array of header-keyed objects), HTML (`<table>`), and XLSX. Supported targets:
  pdf / json / html / xlsx.

## 8. Risks / notes

- **Grid complexity.** `two_dimensional_scrollables` handles virtualization and pinned
  rows / columns, so we avoid a bespoke scroll engine. Column resize / auto-fit is our own
  width state on top of it. Large-file smoothness is Phase 10's job; here we build correct
  behavior on moderate files.
- **Dual-edit sync.** Raw ↔ table re-parse happens on mode switch and save, not on every
  keystroke, to keep it simple and predictable. Documented in the session.
- **PDF non-ASCII.** Same known limit as Phase 5: the built-in PDF font is Latin-1, so a
  bundled Unicode font remains a later polish item; noted, not fixed here.
- **On-device manual pass owed** (recorded in the change log): open a real `.csv` via SAF;
  edit cells; add / delete / reorder rows + columns; dedup; sort / filter / jump; toggle
  raw; insights + chart; export each target; split; merge; overwrite-save; save-as-copy.
