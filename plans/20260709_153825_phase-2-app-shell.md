## 1. What this phase is

Build the frame the user lives in: the theme system, adaptive navigation, the Home /
Recent Files screen, first-run onboarding, and the multi-document tab system (with a
memory-aware cap, swipe between tabs, and optional restore-on-relaunch).

Phase 3 (the shared editor) and Phase 4 (the TXT viewer) are **not built yet**. So each
open tab shows a **placeholder document view** (file name, type, size, and a "viewer
coming soon" note). The tab holds the per-document view state (scroll position, unsaved
flag, read-only flag) so Phase 3/4 can drop the real viewer in with no shell rework.

## 2. What the issue / gap is

- The app still shows the Phase 0 placeholder home (`lib/app.dart` → `_PlaceholderHome`).
- There is no theme system, no navigation, no Home/Recent screen, no onboarding, and no
  tab system. Phase 1 gave the data layer (recents repo, prefs/secure store, device
  memory + `autoTabCap`, config); none of it is wired into a UI yet.

## 3. Design decisions (before coding)

1. **Synchronous settings store at startup.** `keyValueStoreProvider` is a `FutureProvider`
   today, which is awkward for a `Notifier` that must expose settings synchronously (theme
   is read by `MaterialApp` on the first frame). Add a synchronous
   `keyValueStoreSyncProvider = Provider<KeyValueStore>((_) => throw ...)` that `main()`
   overrides after opening the store once at boot. Tests override it with an in-memory
   store. A `KeyValueStore.open()` factory is added for the boot path. The existing
   `FutureProvider` stays for any async callers.
2. **No new packages.** Material 3 dynamic color needs the `dynamic_color` package, which
   the plan says not to add in Phase 2. Use `ColorScheme.fromSeed` as the safe fallback
   (arch §5 allows "dynamic color where available with a safe fallback"). A `TODO` marks
   where dynamic color can be layered in later.
3. **Sepia is a third theme**, not a dark/light flag: a warm light `ColorScheme` +
   background. `ThemeMode`-style enum is app-defined (`AppThemeMode`: light / dark / sepia
   / system) because Flutter's `ThemeMode` has no sepia.
4. **Tab restore persistence uses `KeyValueStore`** (a JSON list under one key), not a new
   DB table — avoids a schema migration. The restore toggle is a bool pref.
5. **Unsaved-edit flag exists but nothing sets it yet** (Phase 3 will). The close guards
   read it now so the "never close an unsaved tab silently" rule is enforced from day one;
   tests set the flag directly.
6. **A minimal Settings screen is built now** with only an **Appearance** section (theme,
   font scale, line spacing) so the theme switch is reachable and testable. It is clearly
   marked as completed later by Phase 11; other sections are not added now.

## 4. Files to change / add

### New — theme (2.1)
- `lib/core/theme/app_theme_mode.dart` — `AppThemeMode` enum (light/dark/sepia/system) +
  labels and pref-string round-trip.
- `lib/core/theme/theme_settings.dart` — immutable `ThemeSettings` (mode, fontScale,
  fontFamily, lineSpacing) with `copyWith` and pref keys.
- `lib/core/theme/app_themes.dart` — `ThemeData` builders for light / dark / sepia from a
  seed color, applying font scale + line spacing to the text theme.
- `lib/core/theme/theme_controller.dart` — `Notifier<ThemeSettings>` that hydrates from and
  persists to `KeyValueStore`; setters for mode/fontScale/fontFamily/lineSpacing.

### New — shell scaffold + nav (2.2)
- `lib/shell/app_shell.dart` — adaptive `Scaffold`: `NavigationRail` on wide viewports,
  `NavigationBar` (bottom) on narrow; destinations Home / Editor / Settings; respects
  system font scaling. Editor destination only enabled when tabs are open.
- `lib/shell/shell_providers.dart` — `selectedDestinationProvider` (which nav area is
  showing) and small shared shell state.

### New — Home / Recent (2.3)
- `lib/shell/home/home_screen.dart` — recents list (name, type icon, path, last-opened),
  one-tap re-open, remove one / clear all, stale entries shown "unavailable" with remove,
  friendly empty state with "Open a file".
- `lib/shell/home/recents_controller.dart` — `AsyncNotifier` over `RecentsRepository` +
  `SafService.isAccessible` to load, mark stale, remove, clear, and record an open.
- `lib/shell/home/file_type_icon.dart` — maps extension/mime to a Material icon.

### New — onboarding (2.4)
- `lib/shell/onboarding/onboarding_controller.dart` — reads/writes an
  `onboarding_complete` bool in `KeyValueStore`.
- `lib/shell/onboarding/onboarding_screen.dart` — short skippable intro; marks complete on
  skip/finish; never shown again.

### New — tabs (2.5, 2.6, 2.7, 2.8)
- `lib/shell/tabs/document_tab.dart` — immutable per-tab model: id, fingerprint, uri,
  displayName, mimeType, size, scrollPosition, isDirty (unsaved), isReadOnly, viewMode,
  lastActiveAt (for LRU).
- `lib/shell/tabs/over_limit_behavior.dart` — enum `closeLruOrAsk` (closeLeastRecentlyUsed
  / ask) + a **pure** `pickLruClosable(tabs, cap)` helper (skips unsaved tabs) for unit
  test 2.6.
- `lib/shell/tabs/tabs_controller.dart` — `Notifier` holding `List<DocumentTab>` + active
  id + resolved cap. Methods: `openFile`, `setActive`, `next`/`prev`, `closeTab`,
  `closeOthers`, `closeAll`, `applyCap`. Enforces the cap on open (LRU close or signal
  "ask"); never closes an unsaved tab silently. Persists the open-tab set for restore.
- `lib/shell/tabs/tabs_persistence.dart` — encode/decode the open-tab set to/from a JSON
  string in `KeyValueStore`; restore skips inaccessible URIs (via `SafService`).
- `lib/shell/tabs/tab_strip.dart` — the tab strip (tap to switch, close button, context
  menu for close others / all; unsaved dot).
- `lib/shell/tabs/tabs_workspace.dart` — the active-tab area with **edge-bound**
  left/right swipe to move prev/next (2.7); hosts the placeholder document view.
- `lib/shell/tabs/placeholder_document_view.dart` — temporary per-tab body until Phase 4.

### New — minimal settings surface (drives 2.1 test)
- `lib/shell/settings/settings_screen.dart` — Appearance section only (theme mode
  segmented control, font scale slider, line spacing); marked "completed in Phase 11".

### Changed
- `lib/main.dart` — `WidgetsFlutterBinding.ensureInitialized()`, open `KeyValueStore`
  once, `runApp` with `keyValueStoreSyncProvider` overridden.
- `lib/app.dart` — `ConsumerWidget`; watch `themeControllerProvider` for `theme` /
  `darkTheme` / `themeMode`; decide first-run (onboarding) vs `AppShell`; remove
  `_PlaceholderHome`.
- `lib/core/storage/key_value_store.dart` — add `KeyValueStore.open()` factory and
  `keyValueStoreSyncProvider`.
- `lib/core/storage/storage_models.dart` — no change expected (reuse `RecentFile`).
- `docs/implementation-progress.md` — flip Phase 2 rows to ✅ and update the summary.

### New — tests (mirror arch §12; each Phase 2 task's "Test")
- `test/core/theme/theme_controller_test.dart` — mode/font round-trip + persistence (2.1).
- `test/shell/theme_switch_widget_test.dart` — switching theme changes the visible scheme
  (2.1).
- `test/shell/adaptive_nav_widget_test.dart` — narrow shows `NavigationBar`, wide shows
  `NavigationRail` (2.2).
- `test/shell/home_screen_widget_test.dart` — populated list renders; empty shows empty
  state (2.3).
- `test/shell/onboarding_widget_test.dart` — shown on first run, hidden after complete
  (2.4).
- `test/shell/tabs_controller_test.dart` — two files → two independent tab states (2.5);
  over-limit picks LRU and an unsaved tab is never silently closed (2.6); restore skips
  inaccessible URIs with a notice (2.8).
- `test/shell/over_limit_behavior_test.dart` — pure `pickLruClosable` unit tests (2.6).
- `test/shell/tab_swipe_widget_test.dart` — edge swipe changes the active tab (2.7).
- Test helpers: a `FakeSafService` (accessible/inaccessible URIs) and an in-memory

## 5. Test / verify plan

- `flutter analyze` → zero issues.
- `flutter test` → all new + existing tests green.
- Manual (owed, noted in the change log, not blocking): run on an emulator to eyeball the
  theme switch, adaptive layout in portrait/landscape, onboarding first-run, and swiping
  between two placeholder tabs. (No emulator run is done as part of this change.)

## 6. Risks / notes

- No editor yet, so tabs show placeholders; the unsaved-edit flag is present but unset
  until Phase 3. This is intentional and called out in the progress notes.
- Making `main()` async and adding the sync store provider is a small cross-cutting change;
  existing Phase 1 providers and tests are untouched (the `FutureProvider` stays).
- Keeping strictly to the "open source only" and "no new packages" rules: no
  `dynamic_color`, no third-party tab/editor widget in this phase.

## 7. Out of scope (later phases)

- Real per-format viewers/editors (Phase 3/4+), full Settings sections (Phase 11), sync
  UI (Phase 12), large-file tab memory release (Phase 10.3), and app-lock/screenshot
  protection (Phase 13). Task 2.8 is the "optional" restore toggle and **is** included.
