# Restore the CSV sort on reopen

**Status:** completed

## The issue

The CSV session remembers a "position", but for a CSV the meaningful state is the
**sort** (which column is sorted, and the direction). Today:

- `persistPosition()` writes only the sort **column** to `csv.pos.<fingerprint>`
  and drops the **direction**.
- Nothing ever reads it back, so reopening a CSV always starts unsorted.

So a CSV never returns to how you had it sorted.

## Files to change

- `lib/formats/csv/csv_document_session.dart`
- `test/formats/csv/csv_document_session_test.dart` (add a round-trip test)

## The fix

1. Store both parts of the sort:
   - keep the column in `csv.pos.<fingerprint>` (`-1` when unsorted),
   - add `csv.dir.<fingerprint>` holding the `SortDirection` index.

2. Add a private `_restoreSort()` that runs inside `load()` (no render timing to
   worry about — sorting is a data operation, so the grid picks it up on first
   build). It:
   - reads the saved column and direction,
   - applies them only if the column is in range for the parsed table and the
     direction is not `none`,
   - otherwise leaves the table unsorted.

3. Call `_restoreSort()` in `load()` after the table is parsed and before the
   session goes to `ready`.

Old saved values that only have the column (no direction key) simply won't
restore until the file is opened and closed once more — acceptable, no migration
needed.

## Testing

- Unit test: sort a loaded session, `persistPosition()`, open a fresh session for
  the same fingerprint, and confirm the sort column and direction come back.
- Confirm a file with no saved sort (or an out-of-range column) opens unsorted
  and does not crash.
