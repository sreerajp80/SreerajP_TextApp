# Fix Pinch-to-Zoom Scrolling Conflict

**Status:** completed

## The issue

When performing a vertical pinch gesture to zoom, the scrollable editor/viewer child view intercepts the gesture as vertical scrolling instead of the parent `PinchToZoomArea` capturing the zoom scale gesture. 

Our initial attempt to solve this by changing the scroll physics to `NeverScrollableScrollPhysics` dynamically did not cancel an already active drag gesture. In addition, the text editors (`TextField` / `EditableText`) do not inherit scroll physics from the parent `ScrollConfiguration`. As a result, the scrollable view continues to scroll when the user pinches.

## Files to change

- [lib/core/editor/pinch_to_zoom_area.dart](file:///l:/Android/SreerajP_TextApp/lib/core/editor/pinch_to_zoom_area.dart)
- [test/shell/tabs/pinch_to_zoom_test.dart](file:///l:/Android/SreerajP_TextApp/test/shell/tabs/pinch_to_zoom_test.dart)

## The fix

1. Wrap the widget tree in `PinchToZoomArea` inside a `NotificationListener<ScrollNotification>`. This listens to scroll notifications bubbling up from any nested scrollable child (e.g. text editors, lists, grids).
2. Store the active child's `ScrollPosition` in `PinchToZoomAreaState` whenever a scroll notification is received.
3. When a second touch pointer is detected (pointer count >= 2), call `jumpTo` on the active child's `ScrollPosition` using its current scroll offset. This programmatically cancels the active drag gesture on the child scroll view immediately.
4. Keep the dynamic `ScrollConfiguration` and `AbsorbPointer` to prevent further scroll inputs and touch actions from starting while the user is actively pinching to zoom.
5. Update unit tests to verify that ongoing scroll drags are correctly cancelled when two fingers are placed down.
