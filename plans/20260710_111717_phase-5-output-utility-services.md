# Plan — Phase 5: Output & utility services

**Status:** completed

Plan for building Phase 5 from [implementation-plan.md](../docs/implementation-plan.md)
(lines 368–413). This phase builds the shared "send it somewhere" services once, in
`lib/core/`, then wires them into the TXT format first (TXT is the only format that
exists so far). Later format phases (6–9) reuse the same services.

---

## 1. What this phase is

Phase 5 adds five shared services:

- **5.1 Share** — share a file or exported output through the Android share sheet.
- **5.2 Zip** — compress a file / output before sharing.
- **5.3 Print** — print a document using its natural output.
- **5.4 Export / convert** — one conversion service; each format declares its targets.
  TXT → PDF / DOCX / Markdown / HTML / plain text must work end to end.
- **5.5 TTS** — read content aloud (English now, Malayalam behind a guided-install
  state machine).

All five are **pure services in `lib/core/`** with a thin platform layer where a
plugin is unavoidable, so the logic stays host-testable (arch §12). They are then
consumed from the TXT toolbar.

---

## 2. What the issue is

The plan's critical path is `… → 4 → 5 → (6–9) …`. Phase 4 (TXT) is done, but there is
no way yet to share, zip, print, export, or read aloud any file. The format phases
(6–9) each depend on Phase 5 (`Depends on: Phase 3, Phase 5`), so these services must
exist and be proven on TXT before the other formats can be built. Right now
`lib/core/share/`, `zip/`, `print/`, `export/`, and `tts/` are empty placeholder
folders.

## 4. Files to be changed / created

### New — core services

- `lib/core/share/share_service.dart` — share text or file bytes; platform call behind
  an injectable `ShareLauncher` so tests verify the built request (MIME + name + bytes)
  without a real share sheet.
- `lib/core/zip/zip_service.dart` — pure `zipEntries(Map<String, Uint8List>) → Uint8List`
  and `unzip(Uint8List) → Map<String, Uint8List>` on top of `archive`.
- `lib/core/print/print_service.dart` — `printPdf(Uint8List, {docName})` behind an
  injectable `PrintLauncher`; tests verify the job is built without printing.
- `lib/core/export/export_target.dart` — `ExportTarget` enum (pdf, docx, html, markdown,
  plainText) + `ExportRequest` / `ExportResult` (bytes + suggested filename + mime) +
  `UnsupportedExportException`.
- `lib/core/export/export_service.dart` — the single conversion service. Holds a map of
  `FormatExporter`s keyed by format id; `supportedTargets(formatId)` and
  `export(formatId, target, TextContent) → ExportResult`; unknown/unsupported target
  throws `UnsupportedExportException` (clean rejection, never a crash).
- `lib/core/export/format_exporter.dart` — `FormatExporter` interface (which targets it
  supports, how it builds each).
- `lib/core/export/pdf_writer.dart` — text → PDF bytes via the `pdf` package (paged,
  wrapped, monospace).
- `lib/core/export/docx_writer.dart` — text → valid minimal `.docx` (OOXML) bytes via
  `archive` (Content_Types, rels, word/document.xml with escaped paragraphs).
- `lib/core/export/html_writer.dart` — text → a simple, escaped HTML document.
- `lib/core/export/txt_exporter.dart` — the TXT `FormatExporter`: supports pdf, docx,
  html, markdown (fenced/plain), plainText.
- `lib/core/tts/tts_service.dart` — `flutter_tts` behind an injectable `TtsEngine`
  interface; a state machine returning `ready` / `needsInstall` / `unavailable` for a
  language; speak / stop; auto-disable with a reason when a voice goes missing.
- `lib/core/tts/tts_state.dart` — `TtsAvailability` enum + `TtsLanguage` (en-US, ml-IN)
  + result types.
- `lib/core/output/output_providers.dart` — Riverpod providers wiring the five services
  (mirrors `editor_providers.dart`).

## 5. Design details

### 5.1 Share
`ShareService.shareFileBytes({name, mimeType, bytes})` writes bytes to the app temp dir
(reuse the `save_tmp` pattern) and calls `ShareLauncher.shareFiles([XFile])`.
`shareText(text, {subject})` for raw text. `ShareLauncher` is an interface with a real
`SharePlusLauncher` impl; tests inject a fake that records the request.

### 5.2 Zip
Pure Dart. `zipEntries` builds an `Archive`, adds `ArchiveFile`s, encodes with
`ZipEncoder`. `unzip` decodes with `ZipDecoder`. No I/O, no platform — fully unit-tested.
Used by "Share as zip" (zip the file/exported output, then hand to `ShareService`).

### 5.3 Print
`PrintService.printPdf(bytes, docName)` calls `PrintLauncher.layoutPdf(...)`
(`printing`'s `Printing.layoutPdf`). For a text document the caller first builds PDF
bytes with the export `pdf_writer`, then prints — "natural output" for TXT is its PDF
rendering. `PrintLauncher` injectable for tests.

### 5.4 Export / convert
- `ExportService` maps `formatId → FormatExporter`. `TxtExporter` registered under
  `"txt"`. `export()` looks up the exporter, checks the target is supported (else throws
  `UnsupportedExportException`), and returns `ExportResult(bytes, suggestedName, mime)`.
- `TextContent { displayName, text, encoding, lineEnding }` is the neutral input so any
  format can feed it later.
- PDF: `pdf` package `Document` with wrapped monospace text across pages.
- DOCX: minimal OOXML — `[Content_Types].xml`, `_rels/.rels`, `word/_rels/document.xml.rels`,
  `word/document.xml` (one `<w:p>` per line, XML-escaped) — zipped with `archive`.
- HTML: escaped `<pre>` document. Markdown/plainText: the text itself (with a suggested
  extension). This satisfies "TXT targets (PDF/MD/DOCX) work end to end".

### 5.5 TTS
- `TtsEngine` interface wraps `flutter_tts` (getLanguages/isLanguageAvailable, setLanguage,
  speak, stop, completion handler). `TtsService` keeps a small state machine:
  - `ready` — the requested language voice is present.
  - `needsInstall` — engine present but the voice (e.g. `ml-IN`) is missing → the UI
    offers the guided install (`INSTALL_TTS_DATA` intent / system TTS settings / Play
    Store) per the plan.
  - `unavailable` — no engine / query failed → the button is hidden, never dead
    (idea Risks).
- English works now. Malayalam sits behind a check; if a previously-ready `ml-IN` voice
  disappears the service auto-disables and reports a reason. The full Settings toggle and
  the install-launcher intents are finished in Phase 11.4 — Phase 5 delivers the state
  machine + speak/stop and an English "Read aloud" in the TXT toolbar.

### TXT wiring
The TXT toolbar overflow gains Share / Share as zip / Print / Export…, and a Read-aloud
toggle (English). Export opens `txt_export_sheet` to choose a target, then Share or
Save-as-copy the produced bytes (Save-as-copy reuses the existing `createDocument` path).

## 6. Testing plan (maps to plan §5 acceptance + arch §12)

- Zip: round-trip bytes equal.
- Share: request carries correct MIME + name + bytes (mocked launcher).
- Print: job built for a sample PDF (mocked launcher).
- Export: TXT→PDF valid (`%PDF-` header, opens); TXT→DOCX valid zip with the text in
  `word/document.xml`, special chars escaped; unsupported target rejected cleanly.
- TTS: state machine returns ready / needs-install / unavailable for mocked voice states;
  auto-disable path covered.
- `flutter analyze` clean; `flutter test` green.

Manual (owed, recorded in the change log like earlier phases): real share sheet, a real
print preview, open an exported PDF/DOCX on device, and hear English read-aloud.
