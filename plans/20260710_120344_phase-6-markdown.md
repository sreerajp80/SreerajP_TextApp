# Plan — Phase 6: Markdown

**Status:** completed

Plan for building Phase 6 from [implementation-plan.md](../docs/implementation-plan.md)
(lines 416–458). This phase adds the **Markdown** format module: a rendered view, a raw
source view, GFM support, a table of contents, front matter, a full editor with a
formatting toolbar and live preview, plus export / metadata / split-merge / a friendly
failure path. It mirrors the TXT module (Phase 4) and reuses every Phase 3 (editor core)
and Phase 5 (share / zip / print / export / TTS) service unchanged.

---

## 1. What this phase is

Phase 6 has five tasks:

- **6.1 Rendered / raw toggle + scroll** — a rendered view (headings, bold/italic, lists,
  tables, blockquotes, code blocks) and a raw source view; toggle between them; remember
  the last scroll position by fingerprint.
- **6.2 GFM + optional extensions** — tables, task-list checkboxes, strikethrough,
  autolinks; optional footnotes/emoji; a `mermaid` block shows as a plain code block (does
  not break); **LaTeX math via `flutter_math_fork`** (user asked to include it now);
  unsupported math degrades to `$$…$$` source.
- **6.3 TOC + jump-to-heading + front matter** — an auto table of contents built from
  headings, tap to jump; internal `#` links jump; YAML front matter shows title / author /
  tags.
- **6.4 Editor + formatting toolbar + live preview** — edit raw source with undo/redo and
  find & replace; live/preview toggle; a toolbar for bold, italic, strikethrough,
  headings, bullet/numbered/task lists, links, inline code / code blocks, blockquotes, and
  tables — each wraps the selection or inserts syntax at the cursor; Save / Save as a copy.
- **6.5 Export / metadata / split-merge / failure path** — MD → HTML / PDF / DOCX;
  metadata (size, modified, word/heading count); split by top-level heading, merge by
  concatenation; corrupt/empty input shows a friendly message, never a crash.

The module follows the TXT pattern exactly: **pure Dart logic classes** (parser walk,
stats, TOC, front matter, toolbar text transforms, split/merge) that are unit-tested with
no device, wrapped by **thin Riverpod/widget UI** that only wires providers and gathers
input. The one genuinely new piece is a **hand-built Flutter renderer** that walks the
Markdown AST — see §3.

## 2. What the issue is

The critical path is `… → 5 → (6|7|8|9) → 10 …`. Phase 5 is done and proven on TXT, but
Markdown files still fall through to the placeholder view: `format_dispatch.dart` already
returns `DocumentFormat.markdown`, yet `tabs_workspace.dart` only special-cases
`DocumentFormat.txt`, so a `.md` file opens as an inert placeholder. `lib/formats/markdown/`
is an empty placeholder folder. Phase 6 fills it and wires it into the shell.

## 3. Package decision (open source only — CLAUDE.md §3.1)

The docs recommend `flutter_markdown`, but that package is **discontinued** by the Flutter
team (its own author points to `flutter_markdown_plus`). Per the user's decision we will
**not** use `flutter_markdown`, `flutter_markdown_plus`, or the stale `markdown_widget`.
Instead:

| Package | License | Publisher | Status | Use |
|---|---|---|---|---|
| `markdown` | BSD-3-Clause | tools.dart.dev (Dart team) | Active (pub ~3 mo) | **Parser only** → Markdown AST (`md.Node` list) |
| `flutter_math_fork` | Apache-2.0 | community fork | Maintained (pub ~13 mo) | LaTeX math rendering (`Math.tex`) |

**We build our own Flutter renderer** (`md_renderer.dart`) that walks the `markdown`
package's AST and maps each node to a Flutter widget. This means:

- No dependency on any discontinued/abandoned rendering widget.
- We reuse the well-tested, actively maintained **parser** rather than re-implementing
  Markdown parsing (which is error-prone and against "never crash on bad input").
- Full control over rendering, links (routed through our own link-warning dialog,
  offline-first), theming (light/dark/sepia), and the mermaid/LaTeX fallbacks.

`flutter_math_fork` is ~13 months since last publish — the best-available pure-Dart LaTeX
renderer, Apache-2.0, functional on Android (our only target). Noted as a mild maintenance
risk; math is isolated behind one small widget so it can be swapped later.

YAML **front matter** is parsed by a tiny built-in key/value/list reader (title / author /
tags only) — no new `yaml` dependency — and degrades gracefully on odd input.

Code-block **syntax coloring** in the rendered view is out of scope for this phase (code
blocks render as styled monospace blocks); it is a later polish item and does not block any
6.x acceptance.

No new Android permissions. Images in Markdown are optional online content (arch §1); local
`file://`/asset images are not resolved (SAF-only, CLAUDE.md §3.3) — a missing image shows a
neutral placeholder, never a crash.

## 4. Files to be changed / created

Module folder: **`lib/formats/markdown/`**, file prefix **`md_`** (mirrors TXT's `txt_`).

### New — pure Dart logic (no Flutter; directly unit-tested)

- `lib/formats/markdown/md_stats.dart` — `MdStats` (words, characters, lines, heading
  count, link count) via a factory over the source + parsed AST. Mirrors `TxtStats`.
- `lib/formats/markdown/md_toc.dart` — `MdToc` / `MdHeading` (text, level, line/anchor);
  builds the TOC list from headings and resolves a `#slug` internal link to a heading.
- `lib/formats/markdown/md_front_matter.dart` — `MdFrontMatter` (title, author, tags, raw
  map) + a splitter that separates leading `---`-fenced YAML from the body. Tiny, tolerant
  parser; returns empty front matter when absent or malformed.
- `lib/formats/markdown/md_source_edits.dart` — pure text transforms for the formatting
  toolbar: given `(source, selectionStart, selectionEnd)`, returns the new source +
  selection for bold, italic, strikethrough, heading (H1–H3), bullet/numbered/task list,
  link, inline code, code block, blockquote, and table insert. This is the class task 6.4's
  test exercises ("each toolbar action produces the expected Markdown around a selection").
- `lib/formats/markdown/md_split_merge.dart` — `MdSplitMerge`: `splitByTopHeading(source)`
  (each top-level `#` heading starts a new part; content before the first heading is its own
  part) and `merge(parts)` (concatenate with a blank line). Round-trip friendly.

### New — rendering (Flutter widgets)

- `lib/formats/markdown/md_renderer.dart` — the hand-built AST→widget renderer. A
  `MarkdownRenderer` that takes the parsed `List<md.Node>` and a style/callbacks bundle and
  emits a `Column` of block widgets. Handles: headings, paragraphs, bold/italic/strikethrough,
  inline code, fenced/indented code blocks, blockquotes, ordered/unordered lists, **task-list
  checkboxes**, **GFM tables**, horizontal rules, links (tap → link-warning), images
  (optional network only, placeholder otherwise), autolinks, `$…$`/`$$…$$` **math** via
  `flutter_math_fork` (bad LaTeX → shows the raw `$$…$$` source), and a `mermaid` fenced
  block → plain code block. Pure widget building; no Riverpod.
- `lib/formats/markdown/md_preview_view.dart` — the scrollable rendered surface bound to the
  session; owns a `ScrollController`, remembers/restores scroll position by fingerprint,
  exposes "scroll to heading" for the TOC and internal links.
- `lib/formats/markdown/md_editor_surface.dart` — raw source editor via `re_editor`'s
  `CodeEditor` (reuses the exact TXT surface pattern: gutter, word-wrap, find panel).
- `lib/formats/markdown/md_live_preview.dart` — the split live-preview layout for edit mode
  (source editor + rendered preview side-by-side on wide screens, stacked/toggle on narrow),
  re-rendering as the source changes.
- `lib/formats/markdown/md_link_warning.dart` — small md-local link-open warning dialog
  (reuses `url_launcher`; same behavior as TXT's, kept local to avoid a cross-format import).

### New — state & shell glue (mirror TXT)

- `lib/formats/markdown/md_document_session.dart` — `MdDocumentSession extends ChangeNotifier`
  (no Riverpod). Loads bytes via SAF, decodes (never throws), holds the `re_editor`
  controllers for the source, tracks a markdown **view mode** (`rendered` / `raw` / `edit`
  with live preview), parses AST/TOC/front matter, exposes `stats`, drives drafts + autosave,
  and saves via `AtomicSaver` + `SafSaveTarget` preserving encoding + line endings. Builds
  metadata with `formatFields` (heading/word/link counts). Mirrors `TxtDocumentSession`.
- `lib/formats/markdown/md_session_manager.dart` — `MdSessionManager` + `mdSessionManagerProvider`
  (map keyed by `tab.id`; `sessionFor`/`peek`/`release`/`retainOnly`). Mirrors the TXT manager.
- `lib/formats/markdown/md_document_view.dart` — the tab body: loading spinner / friendly
  failure view (Retry) / ready view; ready view shows front-matter banner + rendered-or-raw-or
  -live-preview per view mode.
- `lib/formats/markdown/md_toolbar.dart` — action bar: rendered/raw/edit toggle, undo/redo
  (edit), find, save, TOC button, read-aloud, and an overflow menu (find & replace, TOC,
  file info, split, append, share, share-as-zip, print, export). Mirrors `TxtToolbar`.
- `lib/formats/markdown/md_format_toolbar.dart` — the formatting button strip shown in edit
  mode; each button calls an `MdSourceEdits` transform and applies the result to the editor.

### New — output helpers (mirror TXT, reuse Phase 5 services)

- `lib/formats/markdown/md_exporter.dart` — `MarkdownExporter implements FormatExporter`,
  `formatId => 'md'`, targets `{html, pdf, docx, markdown, plainText}`. **HTML** is genuine
  rendered HTML via the `markdown` package (`markdownToHtml`, GFM extension set) wrapped in a
  self-contained offline HTML shell (reusing the `HtmlWriter` shell style). **PDF/DOCX** reuse
  the shared `PdfWriter`/`DocxWriter` over the source text for this phase (structure-faithful
  PDF/DOCX is a later enhancement, noted). `markdown`/`plainText` pass the source through.
- `lib/formats/markdown/md_output_actions.dart` — `MdOutputActions` (`formatId = 'md'`):
  `shareFile`, `shareAsZip`, `printDoc`, `runExport`, `saveExport`, `shareExport`. Mirrors
  `TxtOutputActions`.
- `lib/formats/markdown/md_export_sheet.dart` — target picker → run → share/save. Mirrors TXT.
- `lib/formats/markdown/md_info_sheet.dart` — stats + metadata + `formatFields` rows.
- `lib/formats/markdown/md_save_options_sheet.dart` — encoding + line-ending choose → save.
- `lib/formats/markdown/md_split_merge_actions.dart` — split dialog → per-part
  `saf.createDocument`; append picked `.md` file → edit mode. Mirrors TXT.

### Changed — wiring

- `lib/shell/tabs/tabs_workspace.dart` — add a `DocumentFormat.markdown` branch in
  `_DocumentBody` (→ `MdDocumentView`) and in `_DocumentToolbar` (→ `MdToolbar`).
- `lib/core/output/output_providers.dart` — register `MarkdownExporter` in
  `exportServiceProvider`: `ExportService([const TxtExporter(), const MarkdownExporter()])`.
- `lib/formats/format_dispatch.dart` — (minor) broaden markdown extensions
  (`md`, `markdown`, `mdown`, `mkd`, `mkdn`) and accept `text/markdown` MIME. No structural
  change.
- `pubspec.yaml` — add `markdown` and `flutter_math_fork`.

### New — tests (`test/formats/markdown/`, mirroring `test/formats/txt/`)

- `md_stats_test.dart` — counts on empty / multi-line / heading-heavy input.
- `md_toc_test.dart` — TOC entries + levels; `#slug` resolves to the right heading.
- `md_front_matter_test.dart` — title/author/tags parsed; absent/malformed → empty, no throw.
- `md_source_edits_test.dart` — **each** toolbar transform wraps a selection / inserts at the
  cursor correctly (task 6.4's guard).
- `md_split_merge_test.dart` — split by top heading then merge reproduces the parts/order.
- `md_document_session_test.dart` — load/decode; edit → save round-trip through the atomic
  saver preserving encoding/line endings; corrupt/empty/wrong-encoding → friendly `failed`
  state (task 6.5 failure-path). Reuses `RecordingSafService` + `test/support` helpers.
- `md_renderer_test.dart` — widget: a sample renders headings/table/task-list; a `mermaid`
  block falls back to a code block; bad LaTeX degrades to `$$…$$` source (task 6.2 guard).
- `test/core/export/md_exporter_test.dart` — MD→HTML is valid HTML containing rendered
  markup; MD→PDF starts with `%PDF-`; MD→DOCX is a valid zip; unsupported target throws
  `UnsupportedExportException`.

## 5. The plan for the build (order)

1. **Packages** — add `markdown` + `flutter_math_fork` to `pubspec.yaml`, `flutter pub get`,
   confirm licenses (BSD-3 / Apache-2.0). Broaden `format_dispatch.dart` extensions.
2. **Pure logic first** (host-tested, no device): `md_front_matter.dart`, `md_stats.dart`,
   `md_toc.dart`, `md_source_edits.dart`, `md_split_merge.dart` + their tests.
3. **Renderer**: `md_renderer.dart` (AST walk) + `md_renderer_test.dart` (GFM, mermaid
   fallback, LaTeX degrade). Then `md_preview_view.dart`, `md_editor_surface.dart`,
   `md_live_preview.dart`, `md_link_warning.dart`.
4. **Session + manager**: `md_document_session.dart` (+ test) and `md_session_manager.dart`.
5. **View + toolbars**: `md_document_view.dart`, `md_toolbar.dart`, `md_format_toolbar.dart`.
6. **Output**: `md_exporter.dart` (+ export test), `md_output_actions.dart`,
   `md_export_sheet.dart`, `md_info_sheet.dart`, `md_save_options_sheet.dart`,
   `md_split_merge_actions.dart`.
7. **Wire** `tabs_workspace.dart` + `output_providers.dart`.
8. **Verify**: `flutter analyze` zero issues; `flutter test` green. Record the on-device
   manual pass as **owed** (open a real `.md` via SAF; toggle rendered/raw; edit with the
   toolbar; live preview; TOC jump; front matter; export HTML/PDF/DOCX; split/append) —
   same "owed manual check" convention Phases 4–5 used.

## 6. How we honour the project rules

- **Open source only** — `markdown` (BSD-3) + `flutter_math_fork` (Apache-2.0); no
  discontinued renderer, no Syncfusion. Renderer is our own code (§3).
- **Never crash on bad input** — parser wrapped so corrupt/empty/wrong-encoding lands in a
  friendly `failed`/empty view; bad LaTeX and mermaid degrade to source; missing images show
  a placeholder (CLAUDE.md §3.4).
- **Scoped storage only** — open/save through SAF + `createDocument`; no image/file
  resolution off broad storage (CLAUDE.md §3.3).
- **Atomic saves + preserve encoding/line endings** — reuse `AtomicSaver` + `SafSaveTarget`
  exactly as TXT (CLAUDE.md §3.5).
- **Never lose edits silently** — reuse drafts/autosave + the shell's unsaved-changes close
  guard (already respects the dirty flag) (CLAUDE.md §3.6).
- **Offline-first** — rendered HTML export is self-contained; only optional online is remote
  images/links on explicit tap (CLAUDE.md §3.2).
- **Simple English** — all UI strings, the change log, and comments in plain English.

## 8. Out of scope for this phase (noted, not dropped)

- Structure-faithful **PDF/DOCX** (headings/lists mapped to native styles) — this phase
  exports PDF/DOCX from the source text; HTML is the fidelity path.
- Syntax **coloring inside code blocks** in the rendered view (styled monospace only).
- Full **YAML** front matter (only title/author/tags/simple lists; no anchors/nested maps).
- Large-file streaming — handled cross-cutting in **Phase 10**.
