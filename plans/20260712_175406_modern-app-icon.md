# Modern app icon

**Status:** completed

## Issue

The Android launcher icon is still Flutter's default blue logo. It does not
identify the app as a modern editor for TXT, Markdown, CSV, JSON, and XML files.

## Design direction

Create a simple, original document-and-data symbol with a strong silhouette. Use
the app's indigo Material 3 brand color with a fresh cyan accent. Keep the mark
clear at small launcher sizes. Do not use words, file-extension labels, a
watermark, a mockup, or fine decorative detail.

## Files to change

- `assets/branding/app_icon_master.png` — keep the high-resolution source art.
- `android/app/src/main/res/mipmap-mdpi/ic_launcher.png`
- `android/app/src/main/res/mipmap-hdpi/ic_launcher.png`
- `android/app/src/main/res/mipmap-xhdpi/ic_launcher.png`
- `android/app/src/main/res/mipmap-xxhdpi/ic_launcher.png`
- `android/app/src/main/res/mipmap-xxxhdpi/ic_launcher.png`
- `change_log/<timestamp>_modern-app-icon.md` — record the completed change.

## Plan

1. Generate a polished square icon concept with the built-in image generation
   tool, based on the app's purpose and indigo theme.
2. Inspect the result at full size and at small launcher size. If needed, make
   one focused revision for clarity.
3. Save the chosen high-resolution master under `assets/branding/`.
4. Resize it with high-quality downsampling to Android's standard legacy
   launcher dimensions: 48, 72, 96, 144, and 192 pixels.
5. Confirm every PNG has the correct dimensions and that the Android manifest's
   existing `@mipmap/ic_launcher` reference resolves to the new files.
6. Run relevant project checks and inspect the final icon sheet visually.
7. Write the required change log and set this plan to `completed`.
