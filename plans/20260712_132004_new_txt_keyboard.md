# Open New TXT Documents for Typing
**Status:** completed

## Issue

A newly created TXT document opens in view mode. The editor is read-only in
this mode, so tapping the document does not show the Android keyboard. The user
must tap the pencil button before typing, which is not the expected flow for a
new blank document.

## Files to change

- `lib/shell/create_document_action.dart`
- `lib/shell/open_file_action.dart`
- `lib/shell/tabs/tabs_controller.dart`
- `lib/formats/txt/txt_document_session.dart`
- `lib/formats/txt/txt_session_manager.dart`
- `test/shell/create_document_action_test.dart`
- `test/formats/txt/txt_document_session_test.dart`
- `change_log/<timestamp>_new_txt_keyboard.md`

## Plan

1. Let the shared open-file flow accept an optional initial tab view mode.
2. Mark documents created by the New Document action to start in edit mode.
   Files opened normally or restored from recents will continue to start in
   view mode.
3. Initialize the TXT document session from the tab's requested view mode, so
   the TXT editor is writable as soon as a new document opens.
4. Add tests that confirm a newly created document requests edit mode and that
   the TXT session respects its tab's initial mode.
5. Run formatting, focused tests, and Flutter analysis.
6. Write the required change log and set this plan's status to `completed`.
