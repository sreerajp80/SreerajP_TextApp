## 1. What was built

The full **JSON format module** (`lib/formats/json/`), wired into the shell so a `.json`
file now opens in a real JSON editor instead of the placeholder. It mirrors the TXT
(Phase 4) and Markdown (Phase 6) modules and reuses every Phase 3 (editor core) and Phase 5
(share / zip / print / export / TTS) service unchanged.

Covers all six Phase 8 tasks:

- **8.1 Views** — pretty (colour-coded), tree (collapsible), raw (source), minified
  (single-line), plus an edit mode; toggle between them from the toolbar.
- **8.2 Tree navigation** — expand/collapse one or all, type badges + child counts, copy a
  node's path (`data.users[3].name`), copy value / subtree.
- **8.3 Search + JSONPath** — filter the tree by key/value; a subset JSONPath query sheet;
  re_editor find (line numbers + bracket matching) in the raw/edit views.
- **8.4 Validation + variants + big numbers** — strict validity with the error line; a
  subset JSON Schema validator; format / minify; NDJSON shown as a record list; lenient
  JSONC/JSON5 read with a "Make strict" action that saves strict JSON; large/high-precision
  numbers kept as exact text.
- **8.5 Editing with a pre-save gate** — raw-source editing and tree edits (edit value/key,
  add, delete); a well-formedness gate blocks an invalid overwrite; save-as-a-copy bypasses
  it; chosen indentation + encoding/line ending.
- **8.6 Insights, output, file-level ops** — insights (keys/depth/array sizes/type
  breakdown); export JSON→CSV/YAML/PDF/HTML/JSON; diff two files; print; zip; copy
  full/minified/subtree; metadata; split/merge top-level arrays.

## 2. Key design points

- **Custom lenient JSON reader** (`json_parser.dart` + `json_node.dart`). `dart:convert`
  could not meet three needs at once, so we hand-wrote a tolerant recursive-descent reader
  that: keeps every number as its **exact source text** (so big/high-precision numbers are
  never rounded), records each node's **source span** (for copy-subtree + tree editing),
  preserves key order, tolerates a JSONC/JSON5 subset (comments, trailing commas, single
  quotes, unquoted keys) in lenient mode, and **never throws** (returns a result with a
  1-based line/column). No new package.
- **Colour-coded pretty view** is our own `SelectableText.rich` built from the tree, using
  theme colour roles so it works in light/dark/sepia — no new dependency.
- **Tree editing** is applied as precise **source-span replacements** on the raw text (the
  single source of truth), so undo/redo, dirty tracking, and the save pipeline keep working.
- **Pre-save gate** (`JsonWellFormedGate`) plugs into the Phase 3 `AtomicSaver.save(gate:)`
  seam; `saveAsCopy` skips it (the built-in escape hatch), matching "save anyway as a copy".
- **JSON Schema** and **JSONPath** are deliberate, documented **subsets** (plan §3.5, §8).

## 5. Deviations from the plan (noted)

- **Exporter location.** The plan put `json_exporter.dart` in `lib/core/export/`. It needs
  the JSON parser (which lives in `lib/formats/json/`), so to keep the `core/export` layer
  free of format dependencies the exporter was placed in `lib/formats/json/` instead and
  registered through the DI wiring. It still implements the shared `FormatExporter` interface.
- **Tools sheets.** The plan listed a `json_diff_sheet.dart`; the JSONPath, validate/schema,
  and diff sheets were combined into one `json_tools_sheets.dart` for cohesion.
- **Export targets.** `ExportTarget` was also extended (by parallel Phase 7 CSV work) with
  `json` and `xlsx`; the JSON exporter supports self-export to `json` and rejects `xlsx`.

## 6. Rules honoured

- **Open source only** — no new package (CLAUDE.md §3.1).
- **Never crash on bad input** — the parser never throws; broken/empty/wrong-encoding land
  in a friendly state; invalid JSONPath/schema degrade to a message (CLAUDE.md §3.4).
- **Scoped storage only** — open/save/diff/merge/schema all go through SAF (CLAUDE.md §3.3).
- **Atomic saves + preserve encoding/line endings** — reuse `AtomicSaver` + `SafSaveTarget`,
  now with the JSON well-formedness gate (CLAUDE.md §3.5).
- **Never lose edits silently** — drafts/autosave + the shell close guard; the gate blocks a
  broken overwrite but offers save-as-a-copy (CLAUDE.md §3.6).
- **Simple English** — UI strings, this log, and comments in plain English.
