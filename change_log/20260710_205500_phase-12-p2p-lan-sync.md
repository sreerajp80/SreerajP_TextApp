## What changed

Added offline, same-LAN, device-to-device sync with no server and no internet.
One device **hosts** (sends), the other **joins** (receives). The pairing code is
out-of-band (QR or typed), never on the wire. Every wire message is sealed with
AES-256-GCM; a wrong code fails the tag and the connection is refused. Merge is
**add-only, client-wins** — the receiver never loses data.

### New files (`lib/sync/`)
- `sync_constants.dart` — every tunable and wire literal (code alphabet/length,
  PBKDF2 iterations, salt/nonce sizes, handshake words, bounded-read caps,
  payload caps, timeouts, QR scheme/host/version, category keys, settings
  allow-list, never-sync keys).
- `sync_crypto.dart` — `Random.secure()` bytes; pairing code
  generate/normalize/format/validate; PBKDF2-HMAC-SHA256 key derivation
  (`pointycastle`); AES-256-GCM `encryptWire`/`decryptWire`
  (`base64(nonce(12) || ciphertext+tag)`, via `encrypt`); strict QR URI
  build/parse that rejects foreign/malformed codes; `SecretResealer`
  (device-key → session-key → device-key).
- `bounded_line_reader.dart` — newline reader with a hard per-line byte cap and a
  `closed` future; no unbounded `readLine` (guards slow-loris / giant-line DoS).
- `sync_transport.dart` — `SyncHost`/`SyncClient` connect-then-choose flow
  (salt → HELLO → ACCEPT → hold open → push), single client, timeouts, wrong-code
  handling; app-agnostic (opaque strings only). Plus `localIpv4Addresses()`.
- `payload.dart` — `SyncPayload.build` / `validateAndParse` (hostile-input caps
  and allow-list); pure `mergeRecords` (add-only, client-wins, natural keys) and
  `mergeSettings` (full = apply all, incremental = fill-only).
- `sync_data_access.dart` — `SyncDataAccess` interface +
  `RepositorySyncDataAccess` bridging to the Phase 1 repositories and settings
  store (the only place sync touches app data).
- `sync_provider.dart` — `SyncController` (ChangeNotifier) orchestrating host and
  client flows, provisioning the device key, and Riverpod providers.
- `ui/` — `sync_landing_screen`, `sync_host_screen` (two tabs), `sync_client_screen`
  (scan/manual → waiting → summary), `share_chooser`, `sync_status_chip`,
  `sync_summary_view`.

### Changed files
- `pubspec.yaml` — added `qr_flutter`, `mobile_scanner`, `encrypt`,
  `pointycastle` (all open source; `pointycastle` pinned to `^3.9.1` to match
  `encrypt`).
- `android/app/src/main/AndroidManifest.xml` — added `INTERNET`,
  `ACCESS_NETWORK_STATE`, `ACCESS_WIFI_STATE`, `CAMERA` (local sockets + QR only).
- `lib/core/storage/bookmarks_repository.dart` — added `all()` for export (the
  only touch outside `lib/sync/`, additive).
- `lib/shell/home/home_screen.dart` — added a Sync entry point in the app bar.

### New tests (`test/sync/`)
- `sync_crypto_test.dart` — wire round-trip, wrong key throws, tamper throws, QR
  rejects foreign/malformed/bad-port/bad-version, code validate, `SecretResealer`
  round-trip, PBKDF2 determinism.
- `bounded_line_reader_test.dart` — line splitting/CRLF, over-cap rejection,
  `closed` on drop, buffering order.
- `sync_transport_test.dart` (loopback) — happy path receives the pushed payload,
  wrong code never connects, host keeps listening then accepts a good code,
  `sendToConnectedClient` with no client throws.
- `payload_test.dart` — build filters categories/settings and refuses protected
  keys; `validateAndParse` rejects wrong app / malformed / unknown mode / oversized
  / over-long field / protected settings and drops unknowns; merge add-only and
  settings full vs fill-only.
- `sync_provider_test.dart` — full loopback flow (connect → push → import summary
  + real DB writes) and a wrong-code failure.
- `sync_ui_test.dart` — status chip per phase, share actions gated until
  connected, summary counts.
- `no_secret_logging_test.dart` — asserts `lib/sync` has no logging calls
  (security-rules).

## Verification
- `flutter analyze` — no issues.
- `flutter test` — all 474 tests pass (46 in `test/sync/`).
- Security spot-check: the pairing code never touches the socket (only the salt
  in clear + AES-GCM sealed lines); `Random.secure()` is the only randomness
  source; no logging in `lib/sync`.
