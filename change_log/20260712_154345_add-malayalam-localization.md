# Change log — Add Malayalam localization + in-app language picker

Implements plan: `plans/20260712_152038_add-malayalam-localization.md`.

## What changed

The app was English-only. It now ships a full Malayalam translation and lets the
user pick the language inside the app (System / English / Malayalam), where
**System** follows the device language.

### New files
- **`lib/l10n/app_ml.arb`** — Malayalam translation of every string in
  `app_en.arb`. All ICU placeholders (`{name}`, `{count}`, …) and plural forms
  are preserved. `gen-l10n` reported no untranslated keys.
- **`lib/core/locale/app_locale.dart`** — `AppLocale { system, english,
  malayalam }` enum with `label`, `prefValue`, safe `fromPrefValue`, and
  `toLocale()` (`null` = follow the device).
- **`lib/core/locale/locale_controller.dart`** — a Riverpod
  `Notifier<AppLocale>` that hydrates from and writes to the settings store,
  mirroring `ThemeController`. Preference key `appearance.language`.
- **`test/core/locale/locale_controller_test.dart`** — default = system,
  hydrate, set/persist round-trip, corrupt-value fallback, and `toLocale`
  mapping.

### Changed files
- **`lib/l10n/app_en.arb`** — added 4 keys: `appearLanguage`, `languageSystem`,
  `languageEnglish`, `languageMalayalam`.
- **`lib/app.dart`** — watches `localeControllerProvider` and passes
  `locale: appLocale.toLocale()` to `MaterialApp` (null follows the device).
- **`lib/shell/settings/sections/appearance_section.dart`** — added a language
  `SegmentedButton<AppLocale>` under the theme control, with localized labels.
- **`lib/l10n/app_localizations*.dart`** — regenerated by `flutter gen-l10n`
  (new `app_localizations_ml.dart`; base and English regenerated with the 4 new
  keys).
- **`test/l10n/localization_test.dart`** — added: `ml` is a supported locale,
  and Malayalam strings / placeholders / plurals resolve under `Locale('ml')`.

## Notes
- **Concurrent change:** while this was in progress, a separate change
  (`change_log/20260712_153729_add-english-malayalam-fonts.md`) added the
  `appearMalayalamFontFamily` key. That key was also translated in `app_ml.arb`
  so the Malayalam file stays complete.
- **Proofreading:** the Malayalam text is machine-assisted and should get a
  native-speaker review before release.
- **Out of scope:** syncing the language choice via the "Display settings" sync
  category was not touched.
- **Fallback safety:** any missing `ml` key falls back to English per-key, so
  the UI never shows a blank or crashes (CLAUDE.md §3.4).

## Verification
- `flutter analyze` — no issues.
- `flutter test test/l10n/localization_test.dart
  test/core/locale/locale_controller_test.dart` — all pass.
- Manual two-language check (switch the in-app picker, confirm device fallback)
  is recommended before release.
