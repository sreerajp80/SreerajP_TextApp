# Change log — Add Pinch-to-Zoom in Editor and Viewers

Implements plan `plans/20260716_073226_pinch-zoom.md`.

## The problem

Users could not zoom in or out of the file text (editor or viewer surfaces) using touch gestures (pinch-in/out), limiting accessibility and editing convenience.

## What changed

`lib/core/theme/theme_settings.dart`:
- Expanded the minimum and maximum font scale limits (`minFontScale = 0.5`, `maxFontScale = 3.0`) to give pinch zoom gestures wider zoom levels.

`lib/shell/settings/sections/appearance_section.dart`:
- Adjusted the font size settings slider to have `25` divisions instead of `8` to match the new 50%-300% range in clean 10% steps.

`lib/core/editor/pinch_to_zoom_area.dart` [NEW]:
- Implemented a reusable `PinchToZoomArea` wrapper using `GestureDetector(behavior: HitTestBehavior.opaque)`. It listens to scale gestures and scales the global `themeControllerProvider` font scale dynamically when 2 or more fingers are active on the screen.

`lib/shell/tabs/tabs_workspace.dart`:
- Wrapped the active `_DocumentBody` in `PinchToZoomArea` to enable pinch-to-zoom gestures globally across all file editors and viewers in the workspace.

`lib/shell/tabs/degraded_document_view.dart`:
- Updated `DegradedDocumentView` to fetch and use the provider's `fontScale` and `lineSpacing` when rendering `SelectableText` so that oversized documents scale correctly during pinch-zooms.

`lib/formats/csv/csv_grid.dart`:
- Converted `CsvGrid` to a `ConsumerStatefulWidget`.
- Scaled row height, header height, and row header width dynamically by `appearance.fontScale`.
- Scaled auto-fit column width computations (character widths, minimum, maximum, padding) dynamically by `appearance.fontScale` to ensure grid cells scale proportionally and do not clip when text is zoomed.

`test/shell/tabs/pinch_to_zoom_test.dart` [NEW]:
- Added a widget test `PinchToZoomArea scales the font size on pinch gestures` that simulates multi-touch pinch-in/out scale gestures and asserts the resulting changes in `themeControllerProvider.fontScale`.

`test/shell/tabs/degraded_document_view_test.dart` and `test/formats/csv/csv_grid_test.dart`:
- Wrapped test widgets in `ProviderScope` and provided mock settings overrides to support Riverpod provider consumption.
