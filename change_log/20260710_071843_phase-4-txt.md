## Summary

Delivered the first working file format end to end: a TXT file now **opens →
views → edits → saves** through the shell tabs, wiring together the Phase 3
services (codec, editor, search/replace, atomic saver, drafts, metadata). Before
this phase an open tab only showed a placeholder with the file name; now it shows
a real viewer/editor.

Two open-source packages were added (per the approved plan): **`re_editor`** (MIT)
for the text surface with a line-number gutter, undo/redo, and built-in
find/replace, and **`url_launcher`** (BSD) to open links — always behind a safety
warning the user requested.

`flutter analyze` reports **no issues**. New tests: 6 files covering all five
Phase 4 tasks (pure stats, split/merge round-trip, binary sniff, link
detection/warning, session load/save/dirty/draft/jump/position/encoding, and
toolbar wiring).

### New — TXT module ([lib/formats/txt/](../lib/formats/txt/))
- `txt_stats.dart` — word / character / line counts (4.3).
- `txt_split_merge.dart` — line- and size-based split, concatenating merge;
  `merge(split(t)) == t` (4.5).
- `txt_content_sniff.dart` — "does this look like text?" heuristic (4.4).
- `txt_link_warning_dialog.dart` — URL detection + the **tap-a-link warning**

  consent (4.1, user request).

  crash-recovery drafts + auto-save, saves via the atomic saver preserving
  encoding + line endings, and remembers the reading position by fingerprint.
- `txt_session_manager.dart` — keeps one session per open tab so editor state
  survives switching tabs; disposes a session when its tab closes.
- `txt_editor_surface.dart` — the `re_editor` surface (gutter, word-wrap,
  find panel, app font settings) for both view and edit.
- `txt_find_panel.dart` — Material 3 find/replace bar driving re_editor's
  `CodeFindController` (adapted from its MIT example).
- `txt_document_view.dart` — loading / failed / ready states + draft and
  binary-warning banners (never crashes on a bad file, CLAUDE.md §3.4).
- `txt_toolbar.dart` — view/edit toggle, undo/redo, find, word-wrap, save, and an
  overflow menu (jump-to-line, links, encoding, file info, split, append).
- `txt_save_options_sheet.dart` — pick encoding + line ending, then Save /
  Save as a copy (read-only falls back to copy).
- `txt_encoding_sheet.dart`, `txt_info_sheet.dart`, `txt_links_sheet.dart`,

## Decisions and limitations

- **Editor surface = `re_editor`** (user-chosen). For TXT it owns the live typing
  surface + undo/redo + find; the Phase 3 pure `EditorController`/`FindReplace`
  stay as the model for the structured formats (JSON/XML/CSV) later.
- **Clickable links via a Links sheet, not inline taps.** `re_editor` paints its
  own text, so inline tap-recognizers do not fire. Links are detected and opened
  from a "Links" sheet — always through the warning dialog. Inline tapping can be
  revisited if we move to a custom viewer.
- **Split saves parts one-by-one** through the create-document picker (no bulk
  multi-save yet); **merge appends** one picked file into the editor. The ordering
  guarantee is covered by the pure `TxtSplitMerge` tests.

## Tests
- New: `test/formats/txt/txt_stats_test.dart`, `txt_split_merge_test.dart`,
  `txt_content_sniff_test.dart`, `txt_link_warning_test.dart`,
  `txt_document_session_test.dart` (load / save round-trip / dirty / draft restore
  / read-only / jump-to-line / position restore / word-wrap + view-mode toggles /
  encoding switch).
- Adjusted three shell widget tests (`unsaved_changes`, `read_only_lock`,
  `tab_swipe`) to open a **non-text** file so they keep testing tab mechanics via
  the placeholder rather than the async TXT pipeline.
- The TXT UI is verified at the **session** level rather than by mounting
  `re_editor` in a widget test: its editor surface spawns highlight isolates, and
  its controllers do real file/DB I/O that a `testWidgets` fake-async clock never
  completes — both make widget tests hang. Session tests cover the same behavior
  reliably and fast.
