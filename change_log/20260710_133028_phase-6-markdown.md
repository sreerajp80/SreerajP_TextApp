## 1. What changed (in simple words)

The app can now open, read, edit, and save **Markdown (`.md`)** files. A Markdown
tab shows a **rendered view** (headings, bold/italic, lists, tables, quotes, code
blocks, task-list checkboxes, links, images, and math), a **raw source view**, and
a **source editor** with a formatting toolbar and a live side-by-side preview. It
also gives a table of contents, front-matter (title/author/tags), export to
HTML/PDF/DOCX, file info, and split/append — all reusing the Phase 3 editor core
and Phase 5 output services.

## 2. Key decision — no discontinued package

The design docs named `flutter_markdown`, but that package is **discontinued**. On
the user's instruction we did **not** use it (nor `flutter_markdown_plus` or the
stale `markdown_widget`). Instead:

- **Parsing** uses the actively maintained **`markdown`** package (BSD-3, Dart
  team) — parser only, giving us the Markdown AST.
- **Rendering** is our **own** Flutter widget renderer (`md_renderer.dart`) that
  walks the AST. No dependency on any abandoned rendering widget.
- **Math** uses **`flutter_math_fork`** (Apache-2.0), added per the user's request
  to include LaTeX now. Bad LaTeX degrades to showing the `$$…$$` source.

Both packages are open source (CLAUDE.md §3.1); no Syncfusion.

## 6. How the project rules were met

- **Open source only** — `markdown` (BSD-3) + `flutter_math_fork` (Apache-2.0);
  renderer is our own code; no discontinued package, no Syncfusion.
- **Never crash on bad input** — parser wrapped; corrupt/empty/wrong-encoding →
  friendly `failed`/empty view (tested); bad LaTeX and mermaid degrade to source;
  missing images show a placeholder.
- **Scoped storage / atomic saves / preserve encoding+line endings / drafts /
  unsaved-changes guard** — all reuse the Phase 3 core exactly as TXT does
  (round-trip preserving Windows-1252 + CRLF is tested).
- **Offline-first** — HTML export is self-contained; only optional online is
  remote images/links, opened on explicit tap after a warning.
