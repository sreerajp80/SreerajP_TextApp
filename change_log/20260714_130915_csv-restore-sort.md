# Change log — restore the CSV sort on reopen

Implements plan `plans/20260714_130915_csv-restore-sort.md`.

## The problem

The CSV session saved only the sort **column** (to `csv.pos.<fingerprint>`) and
dropped the **direction**, and nothing ever read it back. So reopening a CSV
always started unsorted — the sort the user had set was lost.

## What changed

`lib/formats/csv/csv_document_session.dart`:
- Added a second key `csv.dir.<fingerprint>` for the sort direction.
- `persistPosition()` now writes both the column and `_sortDirection.index`.
- Added `_restoreSort()`, called inside `load()` after the table is parsed. It
  re-applies the saved column + direction only when the column is still in range
  for the parsed table and the direction is a real sort (not `none`); otherwise
  the table opens unsorted. No render timing is involved — sorting is a data
  operation the grid reads on its first build.

`test/formats/csv/csv_document_session_test.dart`:
- Added a round-trip test: sort a session, persist, open a fresh session for the
  same fingerprint, and confirm the column, direction, and visible row order all
  come back.
- Added a test that an unsorted file opens unsorted.
- Added a test that a saved column out of range for the table is ignored (no
  crash, opens unsorted).

## Note

Old saved values that hold only the column (no direction key) will not restore
until the file is opened and closed once more. No migration is needed.

## Verification

- `flutter analyze lib/formats/csv` — no issues.
- `flutter test test/formats/csv/csv_document_session_test.dart` — all 14 pass.
