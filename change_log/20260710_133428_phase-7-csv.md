# Change Log — Phase 7: CSV

**Implements:** [plans/20260710_133428_phase-7-csv.md](../plans/20260710_133428_phase-7-csv.md)
**Date:** 2026-07-10

## What changed

Added the **CSV** format module: a real data-grid editor with parsing, table
navigation, a raw-text view, read-only insights with a chart, full in-grid
editing, and the shared output / file-level operations. CSV files now open in a
grid instead of the placeholder. It follows the TXT/Markdown/JSON module shape:
pure Dart logic classes (host-tested) wrapped by thin Riverpod/widget UI, reusing
the Phase 3 editor core and Phase 5 output services.

## Packages added (all open source — CLAUDE.md §3.1)

- `csv` ^8.0.0 (BSD-3) — RFC-4180 parse/serialize.
- `two_dimensional_scrollables` ^0.5.3 (BSD-3, flutter.dev) — `TableView.builder`
  with pinned (frozen) rows/columns and lazy cell building.
- `fl_chart` ^1.2.0 (MIT) — the insights bar chart.

## Decisions taken (approved in the plan)

- **XLSX only** for spreadsheet export; the legacy `.xls` binary is out of scope
  (no maintained open-source Dart writer; XLSX supersedes it).
- Insights chart uses `fl_chart`.

## Changed files

- `pubspec.yaml` — added the three packages.
- `lib/core/export/export_target.dart` — added `json` and `xlsx` targets
  (extension/MIME/label). `csv`/`yaml` were already present.
- `lib/core/export/txt_exporter.dart`, `md_exporter.dart` — extended their
  exhaustive switches for the two new targets (they still reject them).
- `lib/core/export/pdf_writer.dart` — added `fromTable(...)` for tabular PDF.
- `lib/core/output/output_providers.dart` — registered `CsvExporter`.
- `lib/shell/tabs/tabs_workspace.dart` — routed `DocumentFormat.csv` to the CSV
  view, toolbar, close-guard saver, and session retain/release.

## Tests

`flutter analyze` clean. New CSV suite green: delimiter/quoting/type inference and
failure paths; table edits + serialize round-trip + dedup; insights aggregates;
filter/sort; split-repeats-header and `merge(split(t)) == t`; CSV→PDF/JSON/HTML/XLSX
and export-selected-rows-only; a session integration test (load, raw toggle
re-parse, filter/sort, cell edit + undo, dedup, CRLF/encoding-preserving save,
read-only save-as-copy); and a grid widget test (render, filter, header sort, raw
toggle). Note: the grid widget test loads via `WidgetTester.runAsync` because the
draft-store DB read cannot complete inside the fake-async zone.

## Known limits / owed

- Manual column resize is auto-fit-by-content only (no drag handle yet).
- Table-mode search is a row filter; find & replace over text is in the raw view.
- PDF export uses the built-in Latin-1 font, so non-ASCII (e.g. Malayalam) does
  not render — a bundled Unicode font remains a later polish item (as in Phase 5).
- One manual on-device pass is still owed (recorded above in the progress doc).
