## What changed

Built the app shell the user lives in: the theme system, adaptive navigation, the
Home / Recent Files screen, first-run onboarding, and the multi-document tab system
(memory-aware cap, edge-bound swipe, optional restore-on-relaunch). Since the editor
(Phase 3) and TXT viewer (Phase 4) do not exist yet, each tab shows a **placeholder
document view**; the per-tab unsaved-edit flag is in place but is not set by anything
until Phase 3.

### New — theme (2.1)
- `lib/core/theme/app_theme_mode.dart` — `AppThemeMode` (light/dark/sepia/system) with
  label + pref round-trip and a safe fallback for corrupt values.
- `lib/core/theme/theme_settings.dart` — immutable `ThemeSettings` (mode, font scale,
  line spacing, font family) with clamped ranges, `copyWith`, and pref keys.
- `lib/core/theme/app_themes.dart` — Material 3 `ThemeData` builders for light / dark /
  sepia from a seed colour, applying font scale + line spacing to the text theme.
  (Dynamic colour left as a documented TODO — needs a package, out of scope this phase.)
- `lib/core/theme/theme_controller.dart` — `Notifier<ThemeSettings>` that hydrates
  synchronously from and persists to the settings store.

### New — shell + navigation (2.2)
- `lib/shell/app_shell.dart` — adaptive frame: `NavigationRail` on wide viewports,
  bottom `NavigationBar` on narrow ones (640px breakpoint); Home / Editor / Settings;
  runs tab-cap resolution and tab restore on startup.
- `lib/shell/shell_providers.dart` — `ShellDestination` + selected-destination notifier.
- `lib/shell/open_file_action.dart` — the one open path (pick → fingerprint → record
  recent → open tab), with friendly SAF error handling.

### New — Home / Recent (2.3)
- `lib/shell/home/home_screen.dart` — recents list, stale/unavailable rows, remove /
  clear-all, empty state, "Open a file".
- `lib/shell/home/recents_controller.dart` — `AsyncNotifier` over `RecentsRepository` +
  SAF accessibility; records opens, removes, clears.
- `lib/shell/home/file_type_icon.dart` — extension/MIME → Material icon.

### New — onboarding (2.4)
- `lib/shell/onboarding/onboarding_controller.dart` — `onboarding.complete` flag.
- `lib/shell/onboarding/onboarding_screen.dart` — skippable 3-page intro; shown once.

### New — tabs (2.5–2.8)
- `lib/shell/tabs/document_tab.dart` — immutable per-tab model (scroll, dirty, read-only,
  view mode, recency).
- `lib/shell/tabs/over_limit_behavior.dart` — over-limit enum + pure `pickLruClosable`
  (skips unsaved tabs).
- `lib/shell/tabs/tabs_controller.dart` — `Notifier<TabsState>`: open/close/others/all,
  next/prev, cap enforcement, dirty guard, restore. `OpenOutcome` signals when the app
  must ask before closing.
- `lib/shell/tabs/tabs_persistence.dart` — save/restore the open set as JSON in
  `KeyValueStore`; restore skips inaccessible URIs and counts them.
- `lib/shell/tabs/tab_strip.dart` — tab strip (switch, close, close-others/all, unsaved
  dot).
- `lib/shell/tabs/tabs_workspace.dart` — active-tab area with edge-bound swipe zones.
- `lib/shell/tabs/placeholder_document_view.dart` — temporary per-tab body.

## Notes / decisions
- **Sync settings store:** `main()` opens `KeyValueStore` once and overrides
  `keyValueStoreSyncProvider`, so the theme is correct on the first frame (no flash).
  The existing `FutureProvider` stays for async callers.
- **No new packages** (CLAUDE.md §3.1): seed-based colour scheme as the safe fallback
  instead of a dynamic-colour package.
- **Widget tests avoid the real DB:** `sqflite_common_ffi` uses a real background isolate
  that never resolves under `testWidgets`' fake-async clock, so Home is stubbed with
  `StubRecentsController` in widget tests (the DB repositories keep their own FFI tests).
