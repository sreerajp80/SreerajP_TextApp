# Change log — Fix Pinch-to-Zoom Scrolling Conflict

Implements plan `plans/20260716_074435_fix-pinch-scroll.md`.

## The problem

When performing a vertical pinch gesture to zoom, the scrollable editor/viewer child view was intercepting the gesture as vertical scrolling instead of zooming. 

Our initial attempt to dynamically alter scroll physics via `ScrollConfiguration` did not affect text editors (since `TextField` and `EditableText` bypass parent scroll configurations). It also did not terminate active drag gestures that had already started.

## What changed

`lib/core/editor/pinch_to_zoom_area.dart`:
- Wrapped the parent tree in a `NotificationListener<ScrollNotification>` to listen to scroll notifications bubbling up from nested scrollable widgets (e.g., text editors, CSV grids, list views) and track the active `ScrollPosition`.
- Updated `Listener.onPointerDown` so that when a second touch pointer is detected (pointer count >= 2), it calls `jumpTo` on the active child's `ScrollPosition` at its current scroll offset. This programmatically terminates the active drag gesture on the child scroll view instantly without disrupting widget states, focus, or resetting scroll offsets.
- Removed the dynamic `ValueKey` on `ScrollConfiguration` to avoid unnecessary element recreation which was causing scroll position resets.

`test/shell/tabs/pinch_to_zoom_test.dart`:
- Added a robust integration-style test `PinchToZoomArea cancels ongoing scroll drag when 2 or more pointers are down` that simulates an active vertical scroll drag and asserts that the drag stops at the exact scrolled offset when a second finger is placed on the screen, and doesn't get reset to `0.0`.
