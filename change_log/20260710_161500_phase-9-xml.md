## 1. What was built

The full **XML format module** (`lib/formats/xml/`), wired into the shell so a `.xml`
file now opens in a real XML editor instead of the placeholder. It mirrors the JSON
(Phase 8) and CSV (Phase 7) modules and reuses every Phase 3 (editor core) and Phase 5
(share / zip / print / export / TTS) service unchanged.

Covers all six Phase 9 tasks:

- **9.1 Views** — pretty (colour-coded), tree (collapsible), raw (source), plus an edit
  mode; toggle between them from the toolbar; scroll position remembered by fingerprint.
- **9.2 Tree navigation** — expand/collapse one or all, tag name + attributes + text
  preview + child counts, copy a node's path (`root/items/item[2]/name`), copy value /
  subtree.
- **9.3 Search + XPath** — filter the tree by tag/attribute/text; an XPath query sheet;
  re_editor find (line numbers + bracket matching) in the raw/edit views.
- **9.4 Format handling + entities** — well-formedness with the error line; format /
  minify; XML declaration + encoding readout; namespace list; comments and CDATA shown as
  their own tree rows; built-in and numeric entities are decoded for display and re-escaped
  on serialize so text is never corrupted. XSD validation is deferred (see §5).
- **9.5 Editing with a pre-save gate** — raw-source editing and tree edits (edit text,
  set/remove attribute, rename, add child, delete, move up/down); a well-formedness gate
  blocks an invalid overwrite; save-as-a-copy bypasses it; chosen indentation + encoding /
  line ending; the declaration/encoding are preserved on a plain save.
- **9.6 Insights, output, file-level ops** — insights (element count, depth, most-common
  tags, attributes per element, namespaces); export XML→JSON/CSV/PDF/HTML/plain text;
  print; zip; copy full/minified; metadata; split by a repeated child element / merge under
  a new wrapper root.

## 2. Key design points

- **`xml` package (MIT).** The one new dependency (`xml: ^6.5.0`, resolved to 6.6.1). It
  parses to a DOM, decodes/re-escapes entities, serializes pretty/minified, and (via its
  `xpath.dart`, part of the same package) evaluates XPath. All open source (CLAUDE.md §3.1).
- **Never-throwing parser wrapper** (`xml_parser.dart`) turns `XmlDocument.parse` into an
  `XmlParseResult` with a friendly message and 1-based line/column. It reads the line from
  the `XmlFormatException` mixin, which covers both syntax errors (`XmlParserException`) and
  tag mismatches (`XmlTagException`).
- **Colour-coded pretty view** is our own `SelectableText.rich` built by walking the DOM —
  no new rendering dependency, works in light/dark/sepia.
- **Tree editing by DOM-mutate-then-re-serialize.** The `xml` DOM has no source offsets, so
  a structural tree edit mutates a parsed document and re-serializes (pretty, chosen
  indent), then flows back through the editor via `applySource` — undo/redo, dirty tracking,
  and the save gate keep working. The raw editor stays the precise, free-form path. This is
  the documented difference from JSON's source-span edits (plan §3.4).
- **Minify strips indentation whitespace** on a copy (whitespace-only text nodes that sit
  beside element children), while keeping significant text, because the package's plain
  serializer otherwise preserves the original indentation text nodes.
- **Pre-save gate** (`XmlWellFormedGate`) plugs into the Phase 3 `AtomicSaver.save(gate:)`
  seam; `saveAsCopy` skips it.

## 5. Deviations from the plan (noted)

- **XSD validation deferred** (plan §3.6, confirmed by the user). The Validate sheet shows
  well-formedness now and states that XSD schema validation is coming in a later update.
  DTD and XSLT stay out of scope. This is a native-only follow-up (Android
  `javax.xml.validation` via a platform channel).
- **CSV export auto-detects the repeated element.** The `FormatExporter.export(target,
  content)` interface takes no extra argument, so `XmlExporter` flattens the root's
  most-common repeated child (via `bestRepeatedTag`) rather than asking in the export sheet.
  The interactive Split action still asks which element to split on.
- **XPath uses the `xml` package's experimental extension.** It is opted into with a local
  `// ignore: experimental_member_use` and degrades to a friendly message on a bad query.

## 6. Rules honoured

- **Open source only** — the one new package (`xml`) is MIT; no Syncfusion (CLAUDE.md §3.1).
- **Never crash on bad input** — the parser never throws; broken/empty/wrong-encoding land
  in a friendly state; invalid XPath degrades to a message (CLAUDE.md §3.4).
- **Scoped storage only** — open/save/split/merge all go through SAF (CLAUDE.md §3.3).
- **Atomic saves + preserve encoding/line endings** — reuse `AtomicSaver` + `SafSaveTarget`,
  now with the XML well-formedness gate; a plain save writes the raw buffer so the
  declaration/encoding are preserved (CLAUDE.md §3.5).
- **Never lose edits silently** — drafts/autosave + the shell close guard; the gate blocks a
  broken overwrite but offers save-as-a-copy (CLAUDE.md §3.6).
- **Simple English** — UI strings, this log, and comments in plain English.

## 7. Owed (manual, on-device)

One manual pass on a device: open a real `.xml` via SAF; toggle pretty/tree/raw; expand/
collapse; copy a path; run an XPath query; validate a broken file; edit text/attributes in
the tree; confirm a blocked invalid save and that the declaration/encoding are preserved;
export JSON/CSV/PDF/HTML; split by an element; merge a file. Same "owed manual check"
convention as Phases 4–8.
